001 : Master's Dissertation Project


View Source Code on GitHub

Transformer models, particularly the Generative Pre-trained Transformer (GPT) series, have changed artificial intelligence, especially in natural language processing (NLP). These models excel in generating human-like text, understanding complex language contexts, and performing various language-related tasks with accuracy. Having been trained on extensive datasets, they offer a cost-effective alternative to custom AI solutions by reducing the need for additional computational resources.

This research investigates the feasibility of repurposing pre-trained GPT models for applications beyond their original design, using the domain of financial predictive analytics as a test-bed. By fine-tuning these models, The study aims to assess the extent to which their advanced language processing capabilities to interpret and analyse data can successfully be used in a different context and predict stock price movements.

The complexity of financial markets - characterised by non-linear relationships, volatile trends, and vast amounts of data from diverse sources — offers a perfect test-bed to research whether fine-tuned GPT models can overcome these challenges, offering reliable financial predictions. The project's broader implications include the potential repurposing of these models for other specialised fields, such as legal analytics and healthcare diagnostics, thereby making advanced predictive analytics more accessible to a wider range of organisations.




System Architecture

The system is built around a dynamic orchestration framework that manages multiple GPT model instances, each with a specialised role in the financial predictive analytics workflow. The architecture consists of several key components, including Python programs, custom scripts, adaptors, and an orchestration program that coordinates the entire process.

Data Collection and Organisation: Python programs interface with external APIs to fetch news data and stock prices. Custom-made Python scripts then organise this data, preparing it for further analysis.

Planner: This program iterates over the data, compiling the information into a structured plan. It functions as a scheduler for the orchestration program, coordinating which data should be fed to the adaptors during each iteration. Like a music sheet for the orchestration program, guiding the sequence and timing of data inputs across the system.

Orchestration: An orchestration program controls the workflow by dictating the actions of the adaptors in each iteration. This program ensures that the system operates in a coordinated manner, overseeing the sequence of data processing, analysis, prediction, and evaluation.

Orchestration files:

Adaptor Functions: Each GPT model is managed by a corresponding adaptor:

  • GPTA Adaptor: Responsible for sentiment analysis of the news data.
  • GPTB Adaptor: Compares sentiment data with stock prices and makes predictions for the next day's prices.
  • GPTC Adaptor: Focuses on analysing historical stock prices and makes independent predictions for the next day's prices.
  • GPTD Adaptor: Receives predictions from GPTB and GPTC, evaluates and compares them, and then makes the final prediction.

Data Flow and Logging: The adaptors fetch and process data as directed by the orchestration program. They communicate with their respective GPT models via API calls, receive the results, format the outputs, and log them accordingly.

Evaluation and Fine-Tuning: After each prediction cycle, the orchestration program initiates an evaluation cycle. During this phase, the models review their predictions by comparing them with the actual stock prices for the next day. This feedback is analysed, formatted, and logged. The system then enters a fine-tuning cycle, where the models learn from any discrepancies in their predictions, improving their accuracy over time.

Iterative Process: The system operates in an iterative manner, continuously refining the models through ongoing cycles of data collection, analysis, prediction, evaluation, and fine-tuning. This iterative approach ensures that the models adapt to changing market conditions and improve their predictive accuracy with each cycle.
